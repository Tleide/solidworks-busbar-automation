# 配电箱铜排连接系统 · SolidWorks 二次开发路线图

> 适用对象：本人执行用 · 技术路线：C#/VB.NET SolidWorks Add-in · 版本 v1.0 · 2026-06-21

---

## 0. 这份文档怎么用

这是一份**给自己看的执行计划**，不是汇报材料。它的作用是：在动手写代码前，先把"做什么、按什么顺序做、每步交付什么、做到什么程度算完成"钉死，避免一头扎进 API 细节后迷失方向。

阅读顺序建议：先看第 1、2 节理解整体定位和架构，再直接跳到第 4 节的分阶段路线图按部就班执行。第 5、6 节是技术储备，遇到对应阶段再查。

核心判断标准（贯穿全文）：

> **几何生成参数化已经能做，不要重复造轮子。二次开发的价值在参数化做不到的四件事——数据/知识层、决策逻辑、拓扑生成、输出自动化。每个阶段的投入都要往这四件事上靠。**

## 1. 项目定位

### 1.1 目标

把"漏保 ↔ 刀熔开关之间的铜排连接系统"从**人工建模**升级为**输入规格 → 自动选型 → 自动校验 → 自动建模 → 自动出文档**的一条自动化链路。

最终形态：输入一份配电箱规格（进线电流、漏保数量与型号、排布方式），程序自动产出一套**经过电气标准校验、可加工、带完整下料/BOM/钻孔文档**的铜排设计。

### 1.2 为什么选 C#/VB.NET 插件路线

| 维度 | VBA 宏 | C#/VB.NET Add-in（本项目选择） |
|---|---|---|
| 上手速度 | 快 | 略慢，需建工程框架 |
| UI 能力 | 弱（用户窗体有限） | 强（WinForms/WPF，可做配置器） |
| 数据层对接 | 难（连数据库/Excel 麻烦） | 强（ADO.NET/EPPlus/SQLite 都顺） |
| 工程化/可维护 | 差（散落的宏） | 好（面向对象、可单元测试、可版本管理） |
| 能力上限 | 有限 | 完整 SOLIDWORKS API + .NET 生态 |

选 C#/VB.NET 的根本原因：本项目的重头戏是**数据层 + 决策逻辑 + 文档输出**，这些恰恰是 VBA 的短板、.NET 的强项。前期投入的工程框架成本，会在 L2 之后持续回报。

> 建议语言用 **C#**（资料、生态、招人都更主流），VB.NET 亦可，二者调用 API 方式几乎一致。

## 2. 核心设计理念

这三条理念决定了整个系统的骨架，必须在动手前内化。

### 2.1 连接接口抽象（最重要）

元件上的特征点不要只当坐标，要升级为**带属性的"连接端子"对象**。每个端子携带：

- 坐标 (X, Y, Z)
- 相位（A / B / C / N / PE）
- 进出方向（IN / OUT）
- 额定电流
- 端子尺寸 / 螺栓规格
- 允许的进线方向（法向）

这样几何就变成了"数据 + 几何"，后续的选型、校验、出 BOM 全部能从端子属性自动推导。**这是从"画图自动化"跨到"知识自动化"的关键一步。**

### 2.2 规则生成 + 可覆盖

不要在"全自动"和"全手动"之间二选一。正确模式是：**规则跑出合理默认值，每个关键参数留可覆盖的口子。**

以汇流排为例：

- 自动推导：长度（最远分支点 X + 端部余量，向上取整到下料模数）、各相 Y（相序 + 间隙算出的节距）、Z 基准、截面规格（按进线电流查载流量）。
- 输入但有默认值：相间节距、铜排标准规格（宽×厚）、端部余量、Z 偏置。

### 2.3 数据驱动，单一数据源

不依赖装配体内部的几何关联（in-context 外部参考在大装配里易失效）。改为**用外部数据（端子坐标表 + 规格库 + 规则表）作为唯一数据源**，程序读取后驱动建模。模型只是数据的"渲染结果"。

## 3. 系统架构分层

```
┌─────────────────────────────────────────────┐
│  UI 层 / 配置器   输入规格、参数覆盖、触发生成      │  ← L4+
├─────────────────────────────────────────────┤
│  输出层   BOM · 下料单 · 钻孔表 · 工程图 · 成本     │  ← L3-L4
├─────────────────────────────────────────────┤
│  逻辑层   选型 · 路径规划 · 间隙/载流量校验 · 拓扑   │  ← L2-L3
├─────────────────────────────────────────────┤
│  几何层   调用 SOLIDWORKS API 建模（钣金/折弯）     │  ← L2
├─────────────────────────────────────────────┤
│  数据层   元件库 · 规格库 · 规则表（唯一数据源）     │  ← L1.5（地基）
└─────────────────────────────────────────────┘
```

开发顺序自下而上：**先打地基（数据层），再几何，再逻辑，最后输出和 UI。** 但价值兑现要尽早——L3 的文档输出最容易见效，不要拖到最后。

## 4. 分阶段路线图

### L1 ✅ 参数化母模型（已完成）

**现状**：用 SWUI 函数 + 全局变量做的参数化模型，效果不错。

**本阶段收尾动作**：

- 把刀熔/漏保上的特征点命名规范固化（A_OUT/B_OUT/C_OUT、A_IN/B_IN/C_IN/N_IN）。
- 整理一份"母模型清单"，记录每个母模型的可变参数、约束、命名约定。

**验收**：命名约定文档化，任何人按约定建的元件都能被后续程序识别。

---

### L1.5 🔨 数据层（地基，优先级最高）

**目标**：把散落在脑子和图纸里的知识，固化成可查询的结构化数据。先用 Excel/CSV 当数据源都行，后续可平滑迁移到 SQLite。

**三张核心表**：

1. **元件库表**：型号 → 各端子的相对坐标 + 属性（相位、进出、额定电流、端子规格、进线法向）。
2. **铜排规格-载流量表**：宽 × 厚 → 额定载流量（含温度、成组敷设降容系数）、标准下料模数、折弯半径。
3. **电气规则表**：额定电压 + 污染等级 → 电气间隙、爬电距离；相序与相色（A黄/B绿/C红/N淡蓝/PE黄绿）。

**技术点**：C# 读 Excel 用 EPPlus 或 ClosedXML；后续上 SQLite 用 System.Data.SQLite。

**验收**：给定一个型号，程序能查出全部端子属性；给定额定电流，能查出推荐铜排规格。

---

### L2 🔨 规则生成器（核心）

**目标**：给定元件布置（一组连接端子），自动生成三段铜排几何。

**关键抽象**：三段铜排统一为"**路径 + 截面**"——一条导体沿 3D 路径（途经点折线）走，截面是某规格。三段的差异只在路径和规格：

- 转接排：刀熔 OUT → 折弯 → 汇流排接入点
- 汇流排：横向直线 + 各分支接出点
- 分支排：汇流排接出点 → 漏保 IN

**实现要点**：

- 用**钣金/折弯特征**建模（不是纯实体扫描），能直接出展开下料长度；折弯半径 R = t～2t。
- 路径规划按第 2.2 的"规则 + 覆盖"模式：自动算默认路径，关键参数可覆盖。
- N 相单独处理（规格不同，走向规律类似）。

**技术点（SOLIDWORKS API）**：`ISldWorks` / `IModelDoc2` / `IFeatureManager` / 钣金相关 `InsertSheetMetalBaseFlange` 等 / 草图与 3D 草图 API / `ISelectionMgr`。

**验收**：给定 3 个漏保 + 1 个刀熔的端子坐标，一键生成三段铜排，折弯展开长度正确。

---

### L3 🔨 校验 + 文档输出（最快见效，优先兑现价值）

**目标**：生成的同时做合规校验，并自动产出制造文档。

**校验项**：

- 电气间隙 / 爬电距离是否满足（查 L1.5 规则表）。
- 铜排截面载流量是否够（查规格表，含降容）。
- 折弯半径、孔到边距离是否合规。

**输出项**：

- **BOM**：每段铜排的相位、规格、长度、数量、铜重。
- **下料单**：展开长度、折弯坐标、数量（直接给车间）。
- **钻孔坐标表**：每个螺栓孔的坐标 + 孔径。
- （进阶）工程图自动出图、铜料成本核算。

**技术点**：Excel 输出用 EPPlus；工程图自动化用 `IDrawingDoc` / 视图与标注 API。

**验收**：一键导出 BOM + 下料单 Excel，数据与模型一致；校验不通过时给出明确提示。

---

### L4 🔨 配置器 UI + 批量

**目标**：脱离手动摆元件，输入高层规格直接出方案。

- WinForms/WPF 做配置器：输入进线电流、漏保数量/型号、排布方式、参数覆盖项。
- 自动布点（拓扑生成）：根据漏保数量和间距规则自动排布端子，而非手摆。
- 批量生成：一次跑多个配置。

**验收**：在 UI 里填规格 → 点生成 → 出完整模型 + 文档，全程无需手动建模。

---

### L5 🔬 优化 + 知识沉淀

**目标**：从"工具"升级为"专家系统"。

- 铜料利用率优化、下料长度标准化、降本。
- 把公司/老师傅的设计标准持续沉淀进规则库，让新人也能产出资深水平设计。
- （远期）与 PDM/ERP/订单数据打通。

**验收**：方案在满足全部电气/工艺约束下，铜料用量较人工设计有可量化下降。

## 5. 技术学习路径（C# + SOLIDWORKS API）

按需查阅，对应阶段再深入。

1. **C# 基础**：类与接口、集合、LINQ、文件/Excel 读写（EPPlus）。
2. **SOLIDWORKS API 入门**：
   - 开发环境：Visual Studio + SOLIDWORKS API SDK，建 Add-in 工程（实现 `ISwAddin`）。
   - 核心对象模型：`ISldWorks`（应用）→ `IModelDoc2`（文档）→ `IPartDoc`/`IAssemblyDoc`/`IDrawingDoc`。
   - 早期调试技巧：先用**宏录制**看 API 调用，再翻译成 C#。
3. **建模相关 API**：草图、3D 草图、特征树 `IFeatureManager`、钣金折弯、配置 `IConfiguration`、方程式/全局变量 `IEquationMgr`。
4. **装配与选择**：`ISelectionMgr`、配合 `IMate2`、遍历特征点/参考几何。
5. **工程图 API**：视图、尺寸、表格（BOM）。
6. **数据持久化**：先 Excel（EPPlus），后 SQLite。

> 学习方法：每学一个 API 就用它解决路线图里对应阶段的一个小问题，避免空学。

## 6. 关键技术难点与对策

| 难点 | 对策 |
|---|---|
| 外部参考（in-context）易失效 | 用外部数据当唯一数据源，程序驱动建模，不靠装配内几何关联 |
| 钣金展开长度精度 | 用 SOLIDWORKS 钣金折弯系数表，与车间实际工艺对齐 |
| 折弯路径自动规划复杂 | 先支持典型走向（规则 + 覆盖），非标工况留手动覆盖口子 |
| N 相规格特殊 | 数据层单列 N 相规格，生成器按相位分支处理 |
| 电气间隙合规 | 间隙/爬电做成规则表，生成时校验，不合规即报警 |
| API 调试效率低 | 善用宏录制 + 日志，先跑通最小闭环再扩展 |

## 8. 给自己的提醒

1. **别在 L2 重复参数化已经做好的事**。几何生成不是重点，数据/逻辑/输出才是。
2. **每个阶段都要有可交付物**，哪怕只是一个能跑的小工具，避免长期看不到产出。
3. **数据层先行**。地基没打好，上面写的逻辑全是临时硬编码，迟早返工。
4. **规则 + 覆盖**贯穿始终，既要自动化，也要给非标工况留口子。
5. 电气合规（GB 7251 / IEC 61439 间隙与爬电）不是可选项，从 L3 起就内建校验。
